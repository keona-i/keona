2. Semester - Full Stack Development

Ein Spiel, um Full Stack zu verstehen

Die Aufgabe bestand darin ein Full Stack - Projekt zu coden. Das Backend sollte mit den Frontend verbunden werden, dabei musste mindestes ein LED-Streifen eingebettet werden. Als Projekt wählte ich das Spiel Vier Gewinnt.



Prototyping

Der Arduino Uno wird mit Javascript betrieben und steuert so die LED's. Im Code wurden die einzelne Pixel mit drei faktoren 0,1,2 festgelegt, dass bedeutet die Pixel kann frei sein oder mit dem Chip vom Spieler 1 besetzt oder es wird mit dem Chip vom Spieler 2 besetzt. Wird im Interface ein Button angeklickt von den jeweiligen Spielern, fällt der Chip in form der LED runter. Druch die Funktion drop und applyGravit wird so die Schwerkraft simuliert, damit sieht es aus wie im echten Spiel. Die zwei Spielern haben unterschiedliche Farben Spieler 1 hat die farbe Rot & Spieler 2 hat die farbe Grün. Mit dem Reset-Button können das Spiel, wieder von vorne angefangen werden.





Full Stack

Im Frontend wurde mit VUE & Bootstrap gearbeitet, um schnell und ohne viel CSS Kenntnisse das Interface zu gestalten. Das Frontend ist mit dem Backend verbunden, durch Socket ist es überhaupt möglich das Interface anzuklicken & so ändert sich die LED Spielfläche. Das bedeutet das im socket-io die Daten in Echtzeit übertragen werden, also die Kommunikation zwischen Interface und Server. Im Backend wurde die Funktionen in JavaScript geschrieben, das wiederum ist mit dem Arduino Uno verbunden, wo auch die LED angepint sind.





Interface

Das Interface kann auf Website aufgerufen werden oder auch am Smartphone mit einen QR-Code.
